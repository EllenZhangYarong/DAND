Data Loading and Processing

The function get_dataloader should transform image data into resized, Tensor image types and return a DataLoader that batches all the training data into an appropriate size.

Nice job resizing the images and creating a DataLoader. The batch size is ideal for this architecture.

Pre-process the images by creating a scale function that scales images into a given pixel range. This function should be used later, in the training loop.

Images are scaled correctly. We are using tanh activation at the last layer of generator, so the pixels in the generated images are in the range of -1.0 to 1.0. Scaling the input images (real images) also into the same range will make the training more effective and faster.

Build the Adversarial Networks

The Discriminator class is implemented correctly; it outputs one value that will determine whether an image is real or fake.

Good:

  • Used a sequence of convolutional layers. 3-4 conv layers are sufficient .
  • Used batch normalization starting from second layer.
  • Used strided convolution instead of max pooling to downsample, making the network to learn its own pooling function.
  • Leaky relu and batch norm promote healthy gradient flow.

Important Suggestion:

  1. Add dropout layers right after each leaky ReLU. See example here: https://github.com/udacity/deep-learning-v2-pytorch/blob/master/gan-mnist/MNIST_GAN_Solution.ipynb This will bring down the generator loss lower than discriminator loss.
    If you look at the plot below, you can see that the generator loss (orange line) gets higher as the training progress.


What we need is the opposite, the generator loss should come down and should be maintained lower than discriminator. That's the importance of dropout here. It will reduce over fitting.

  1. Remove the 5th layer. Above 4 layers, with this 32x32 image its gets overfit easily, and more epochs won't help it.
  2. The self.bnormx layers are redundant here, you already added batchnorm inside the conv()
  3. With the above changes, it better to remove the sigmoid() before the final fully connected layer.

The Generator class is implemented correctly; it outputs an image of the same shape as the processed training data.

Good:

  • Used a series of strided covolutional transpose layers.
  • Used tanh at the output to return in the range -1 to 1.

Suggestion:

  • Use Relu activation instead of leaky relu as suggested in DCGAN paper(leaky relu for discriminator and relu for generator).
  • Make the generator at least one more layer bigger than discriminator, here reduce discriminator layer.
  • Try different values of z_size between 64-256 for varied types of faces.

Optimization Strategy

The loss functions take in the outputs from a discriminator and return the real or fake loss.

Important suggestion:

  • Use label smoothing. This will prevent discriminator from being too strong by encouraging it to estimate soft probabilities.
    This is done for real_loss()
    Example: labels = torch.ones(batch_size) *0.9

More info:
http://www.inference.vc/instance-noise-a-trick-for-stabilising-gan-training/

There are optimizers for updating the weights of the discriminator and generator. These optimizers should have appropriate hyperparameters.

Adam optimizer is a right choice. The learning rate and batch size are appropriate. Learning rate in the range of 0.0002-0.0008 works well with a batch size of 16-32. The batch size and learning rate are linked. The smaller batch size should go with low learning rate.
